## 📌 3. 브랜치 및 커밋 전략 (Git Workflow)

### 브랜치 전략 (Branch Strategy)

- **`main`**: 실제 배포되는 브랜치
- **`dev`**: 개발 통합 브랜치
- **`feature/*`**: 새로운 기능 개발 브랜치
- ➡️ _기능은 `feature/_`브랜치에서 개발하여`develop`브랜치에 모은 후, 최종적으로`main` 브랜치에 배포합니다.\*

### 커밋 컨벤션 - 작게 나누어서 자주! (Commit Convention)

- **`Feat`**: 새로운 기능 추가
- **`Fix`**: 버그 수정
- **`Docs`**: 문서 수정
- **`Style`**: 코드 포맷팅, 세미콜론 누락 등 (코드 변경 없는 경우)
- **`Refactor`**: 코드 리팩토링
- **`Test`**: 테스트 코드 추가
- **`Chore`**: 빌드 업무 수정, 패키지 매니저 설정 등
