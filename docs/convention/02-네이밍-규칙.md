## 📌 2. 네이밍 규칙 (Naming Conventions)

### 파일 및 폴더 이름 (File & Folder)

- **컴포넌트 파일 (`.tsx`)**: 파스칼 케이스 (PascalCase) ➡️ `Button.tsx`, `ProductCard.tsx`
- **일반 훅, 함수, 유틸 파일 (`.ts`)**: 카멜 케이스 (camelCase) ➡️ `useAuth.ts`, `formatDate.ts`
- **폴더 이름**: 카멜 케이스 (camelCase) ➡️ `productDetail`, `searchResults`
- **이미지 에셋 파일**: 스네이크 케이스 (snake_case) ➡️ `check_white.svg`, `landing_bg_large.png`

### 컴포넌트 & 타입 명명법 (Component & Type)

- **컴포넌트 함수명**: 파스칼 케이스 (PascalCase) 및 화살표 함수 통일
  ```tsx
  const ProductList = () => { ... };
  ```

````

* **인터페이스 & 타입 (`interface`, `type`)**: 파스칼 케이스 (PascalCase) / 접미사 생략 규칙
```typescript
interface User {}
type ButtonVariant = 'primary' | 'secondary';

````

- **Props 타입**: `컴포넌트이름 + Props` 구조로 통일

```typescript
interface ButtonProps {}
```

### 함수 & 이벤트 핸들러 (Function & Event)

- **내부 이벤트 핸들러**: `handle + 행위/동사` ➡️ `handleClick`, `handleSubmit`
- **자식에게 넘겨주는 이벤트 Props**: `on + 행위/동사` ➡️ `onClick`, `onSubmit`
- **구현 구조:**

```tsx
const handleSearchSubmit = () => { ... };
<SearchForm onSearch={handleSearchSubmit} />

```

### 변수 및 상태 명명법 (Variables)

- **Boolean 변수/상태**: `is`, `has`, `should`, `can` 접두사 사용
- `is + 상태`: `isLoggedIn` (로그인됨), `isLoading` (로딩중)
- `has + 존재`: `hasToken` (토큰 있음), `hasError` (에러 있음)
- `should + 동사`: `shouldShowModal` (모달 필요함)

- **배열 (Array) 변수**: 복수형(`s`) 또는 `List` 명시 ➡️ `products`, `productList`
- **전역 상수 (Constant)**: 전체 대문자 + 스네이크 케이스 (UPPER_SNAKE_CASE) ➡️ `const API_BASE_URL = '...'`
